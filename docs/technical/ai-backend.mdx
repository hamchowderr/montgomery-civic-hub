## Overview

Montgomery Civic Hub implements a dual AI backend — a unique architecture that runs both a traditional streaming endpoint and an AG-UI agent protocol simultaneously.

## Streaming Endpoint

**Route**: `POST /api/chat/[portal]`

A Claude tool-use loop that:
1. Receives user message + portal context
2. Runs up to 5 tool-call iterations (non-streaming, server-side)
3. Streams the final text response to the client
4. Persists the conversation to Convex

## AG-UI Agent (CivicAgent)

**Route**: `POST /api/copilotkit`

The `CivicAgent` class extends `@ag-ui/client AbstractAgent` and:
1. Emits events via RxJS Observable streams
2. Supports tool-use visibility in the UI
3. Integrates with CopilotKit for rich chat experiences
4. Handles all four portal agents from a single endpoint

## System Prompts

Portal-specific prompts are centralized in `lib/ai/prompts.ts`. Each prompt configures:
- **Tone** — Professional, friendly, academic, or technical
- **Focus areas** — Which datasets and topics to prioritize
- **Formatting** — How to structure responses for the audience
