## ArcGIS FeatureServer

The primary data source is Montgomery's public ArcGIS FeatureServer, providing 40+ datasets including:

- **Crime & Safety** — Incident reports, arrests, calls for service
- **Permits & Licenses** — Building permits, business licenses, zoning
- **Infrastructure** — Roads, utilities, public facilities
- **Demographics** — Census data, population, housing

All queries hit live endpoints — no stale data. The `lib/arcgis-client.ts` module manages dataset URLs and browser-safe queries without API keys.

## Bright Data Web Scraping

Supplementary data is scraped from web sources via the Bright Data MCP integration, providing:
- News and announcements
- Public meeting records
- Supplementary context for AI chat responses

## Caching

| Cache | TTL | Storage |
|-------|-----|---------|
| ArcGIS query results | 15 minutes | Convex `arcgis_cache` table |
| Web scrape results | 1 hour | Convex `scraped_cache` table |
| Client-side | React Query defaults | Browser memory |

## Year Filtering

The `YearFilterProvider` context enables filtering all data queries by year range (2018–present). The `buildWhereClause()` utility generates ArcGIS-compatible SQL WHERE clauses for date-filtered queries.
