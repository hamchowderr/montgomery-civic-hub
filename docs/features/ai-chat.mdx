## Overview

Each portal includes an AI chat interface powered by Claude claude-sonnet-4-20250514 with tool use. Users ask questions in natural language and receive answers backed by real Montgomery city data.

## How It Works

1. **User asks a question** — e.g., "What are the crime trends in District 4?"
2. **AI agent processes** — Determines which tools to call based on the query
3. **Tools execute** — `arcgis_query` fetches from ArcGIS FeatureServer, `brightdata_search` scrapes web sources
4. **Response streams** — Results are formatted and streamed back in real-time
5. **Message persisted** — Conversation saved to Convex for session continuity

## Tools

| Tool | Source | Purpose |
|------|--------|---------|
| `arcgis_query` | Montgomery ArcGIS FeatureServer | Crime data, permits, infrastructure, demographics |
| `brightdata_search` | Bright Data MCP | Web scraping for supplementary information |

## Portal Personality

Each portal's AI has a distinct personality and focus:

- **Resident**: Friendly, safety-focused, uses plain language
- **Business**: Professional, permit/license-oriented, includes actionable next steps
- **City Staff**: Technical, operations-focused, budget-aware
- **Researcher**: Academic, data-rich, includes methodology context

## CopilotKit Integration

The chat system uses CopilotKit with the AG-UI agent protocol, enabling:
- Observable event streams via RxJS
- Tool-use visibility in the UI
- Portal context awareness via `useCopilotReadable()`
- Real-time streaming responses
