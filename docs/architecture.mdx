## System Overview

Montgomery Civic Hub uses a modern full-stack architecture with dual AI backends, client-side data fetching, and real-time persistence.

```
Portal UI → CopilotProvider → CopilotKit Runtime (/api/copilotkit)
                                → CivicAgent (AG-UI AbstractAgent)
                                  ├→ arcgis_query → ArcGIS FeatureServer
                                  └→ brightdata_search → Bright Data MCP
                                → Convex (persist messages + cache results)
```

## Dual AI Backend

Two parallel AI systems handle chat interactions:

1. **Streaming Endpoint** — `POST /api/chat/[portal]` routes through a Claude tool-use loop (max 5 iterations) and streams the final response to the client.

2. **CopilotKit + AG-UI Agent** — `POST /api/copilotkit` routes through the CivicAgent class, which extends the AG-UI `AbstractAgent` and emits events via RxJS Observables.

Both systems share the same tools (`arcgis_query`, `brightdata_search`), system prompts, and Convex persistence layer.

## Data Flow

### Chat Queries
User asks a question → AI agent determines which tool to call → tool queries ArcGIS or Bright Data → results formatted and streamed back → message persisted to Convex.

### Dashboard Data
Portal components fetch data client-side via `lib/arcgis-client.ts` directly from Montgomery's ArcGIS FeatureServer. This bypasses server-side IP restrictions that block cloud providers.

### Caching Strategy
- **ArcGIS cache**: 15-minute TTL in Convex
- **Web scrape cache**: 1-hour TTL in Convex
- **Client-side**: React Query manages frontend caching

## Key Design Decisions

<AccordionGroup>
  <Accordion title="Why client-side ArcGIS queries?">
    Montgomery's GIS server blocks requests from cloud IPs (Convex, Vercel). Browser-direct queries work reliably and reduce server load.
  </Accordion>
  <Accordion title="Why dual AI backends?">
    The original streaming endpoint provides simple, fast responses. The AG-UI agent protocol enables richer interactions with observable event streams, tool-use visibility, and CopilotKit integration.
  </Accordion>
  <Accordion title="Why Convex over Supabase?">
    Convex provides real-time reactivity out of the box, which powers live chat persistence and cache invalidation without additional WebSocket setup.
  </Accordion>
</AccordionGroup>
